Build an Oracle CRUD API in Go

A Hands-On Training Guide for Junior Developers

Using godror, OpenAPI, and Best Practices

Learning Objectives

  • Use godror to connect Go apps to Oracle DB
  • Structure a small service (db, repo, api) using packages
  • Serve OpenAPI docs and use Postman for testing
  • Configure a connection pool and avoid Oracle OCI pitfalls
  • Hot-reload SQL queries from a properties file without restarting

Prerequisites

Required Software

  • Go 1.23+ (check with go version)
  • Git and VS Code (or your editor of choice)

Oracle Database Access

Choose one option:

  • Local Oracle XE (Docker or native install), or
  • Remote Oracle instance you can reach (host/port/service)

Oracle Client Setup for godror

Choose one option:

  • Native: Install Oracle Instant Client (Basic). Add its bin to PATH (Win) / LD_LIBRARY_PATH (Linux) / DYLD_LIBRARY_PATH (macOS)
  • Thin mode: Consult godror docs to enable driver thin mode (if supported)

Path A: Run This Project

  1. 1

    Clone and Open

    git clone <your-repo-url> && cd godror-crud

    Open in VS Code

  2. 2

    Configure Database

    Edit config/application.properties and set:

    • db.service, db.username, db.password
    • db.server, db.port, db.timezone

    Optional pool tuning (recommended):

    • db.pool.max_open = 20
    • db.pool.max_idle = 10
    • db.pool.conn_max_lifetime = 30m
    • db.pool.conn_max_idletime = 5m

Best Practices and Tips

Connection Pooling

Keep db.pool.max_idle > 0 to avoid OCI reconnect churn. Consider driver session pooling by setting DSN params: standaloneConnection=0 poolMinSessions=1

Timezone Configuration

Set DSN timezone to match DB SYSTIMESTAMP (often +00:00) to avoid warnings and timestamp confusion

Error Handling

Check logs for config reload issues; the app keeps last good SQL on parse failures

BLOB/JSON Handling

For attributes_blob, send base64; for large text, prefer CLOB

Troubleshooting

Issue Solution
Discrepancy between SESSIONTIMEZONE and SYSTIMESTAMP Set timezone="+00:00" (or your local zone) in the DSN
OCI library load errors Verify Instant Client install and PATH/LD_LIBRARY_PATH
Build status 0xc000013a on Windows Usually indicates the process was interrupted (debugger stopped)
Permission/auth errors Confirm Oracle credentials and network reachability

Project Architecture

Key Files in This Repository

  • main.go — wiring and server startup
  • internal/db/config.go — DB config, DSN, pool application
  • config/application.properties — DB settings + SQL queries (hot-reloaded)
  • internal/config/properties.go — query reload manager
  • internal/config/app.go — load DB config from properties
  • internal/repo/product_repo.go — CRUD SQL
  • internal/api/handlers.go — HTTP handlers and routes
  • internal/docs/openapi.yaml — API schema (import into Postman)
